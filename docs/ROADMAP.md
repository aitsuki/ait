# ait Roadmap

ait 的路线图以“小步发布、先解决真实使用阻力”为原则。这里列的是当前方向，不是承诺清单；如果用户反馈暴露出更紧急的问题，版本目标可以调整。

## v0.1.x: 早期可用性

目标：减少用户第一次安装和日常使用时的阻力。

- v0.1.2: 开机自启设置。
- v0.1.2: 在设置或托盘入口显示当前版本号。
- v0.1.x: 增加日志和诊断入口，方便用户反馈问题。
- v0.1.x: 增加首次启动提示，说明默认快捷键和托盘入口。
- v0.1.x: 改进常见失败场景的错误提示。

## v0.2.x: 更新和发布可信度

目标：让用户更容易拿到新版本，并逐步降低 Windows 安全提示带来的不信任感。

- 检查更新，并提示用户有新版可用。
- 从应用内打开 GitHub Releases 最新版页面。
- 研究代码签名证书的成本、审核要求和发布流程。
- 改进安装器体验，包括安装后启动、卸载入口和版本信息。

## v0.3.x: 翻译体验打磨

目标：提升核心翻译流程的稳定性和舒适度。

- 继续优化翻译窗口布局、状态和操作细节。
- 评估是否需要可选的翻译历史。
- 增强翻译服务配置能力。
- 继续打磨快捷键、托盘菜单和窗口行为。

## Later

这些事项价值较高，但应该在基础体验和发布可信度更稳定后再做。

- 自动下载并安装更新。
- 对正式发布的安装器和可执行文件做代码签名。
- 更完整的崩溃和错误反馈机制。
- 更多平台支持。

## 当前推荐下一步

下一版建议定为 v0.1.2，范围控制在：

- 开机自启设置。
- 当前版本显示。

这两个功能都直接改善日常使用，并且实现风险较低，适合作为 v0.1.1 之后的下一个小版本。
