# API Key 明文显示设计

## 背景

设置窗口当前不会把 API Key 明文回填到输入框。非 Google 翻译配置如果已经保存过 API Key，界面只显示占位文案“已保存”；因为该输入框使用密码样式，用户实际看到的是三个掩码字符。这个行为能避免明文暴露，但用户无法查看或复制已保存的 Key，也容易误解为只保存了三个字符。

## 目标

- 用户可以在设置窗口中主动查看已保存的 API Key 明文。
- API Key 默认仍然隐藏，避免打开设置窗口时直接暴露敏感信息。
- 默认隐藏状态显示固定长度的掩码，占位效果比三个星号更清晰。
- 保存到配置文件时仍只保存 DPAPI 保护后的密文，不保存明文。
- 日志、view model 和测试输出不记录 API Key 明文。

## 非目标

- 不增加复制按钮。用户可以显示明文后手动选择复制。
- 不改变 Google 内置免 Key 翻译配置的行为。
- 不实现跨 Windows 用户或跨设备的密钥迁移。
- 不显示真实 API Key 长度。

## 交互

非 Google profile 的 API Key 输入框旁新增一个“显示/隐藏”按钮。

当 profile 没有保存 API Key 时，输入框为空，按钮不可用或点击后不做解密动作。

当 profile 已保存 API Key 时，输入框默认显示固定占位 `********`，输入框仍使用密码样式。用户点击“显示”后，应用使用当前 profile id 对应的 `SecretStore("ait-translator-profile-{id}")` 解密 `encrypted_api_key`，把明文写入输入框，取消密码样式，并把按钮文案切换为“隐藏”。

用户点击“隐藏”后，应用清除控件里的明文，恢复固定占位 `********`，重新启用密码样式，并把按钮文案切换为“显示”。

切换 profile 时，如果之前的输入框正在显示明文，窗口必须丢弃旧明文，并按新 profile 的状态重新显示空值或固定占位。

## 保存规则

保存设置时根据 API Key 输入框内容决定如何处理 `encrypted_api_key`：

- 内容等于固定占位 `********`：用户未修改 API Key，保留当前 profile 已有的 `encrypted_api_key`。
- 内容为空：用户清空 API Key，保存为 `None`。
- 内容为其他字符串：作为新的 API Key，用当前 profile id 对应的 `SecretStore` 重新 DPAPI 加密后保存。
- Google profile：继续清空网络字段和 `encrypted_api_key`。

## 错误处理

点击“显示”时如果 DPAPI 解密失败，窗口弹出“API Key 读取失败，请重新保存接口配置。”，输入框保持隐藏占位状态，不修改配置。

如果保存时加密失败，沿用现有保存失败弹窗，配置不落盘。

## 测试

测试覆盖：

- View model 仍只暴露 `has_api_key`，不暴露密文或明文。
- 已保存 Key 的 profile 默认 API Key 文本为固定占位 `********`。
- 保存固定占位时保留原密文。
- 保存空字符串时清除 Key。
- 保存新字符串时替换为新的加密结果。
- 显示/隐藏状态转换不会把明文留在隐藏状态的输入框中。
- Google profile 仍不显示 API Key 字段并清空相关网络字段。
