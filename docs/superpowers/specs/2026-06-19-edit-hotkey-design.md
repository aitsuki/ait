# 修改快捷键设计

## 背景

设置窗口已经显示“快捷键”字段，并且保存时会把文本写入 `AppSettings.hotkey`。主程序启动时会读取该字段并通过 `RegisterHotKey` 注册全局快捷键。

当前缺口有两个：

- 设置窗口里的快捷键字段是普通文本框，用户可以输入任意内容。
- 主程序收到设置保存消息后只重载配置和刷新翻译配置列表，不会把运行中的全局快捷键切换到新值。

这会导致用户输入非法快捷键、保存后不立即生效，或者重启时才发现快捷键注册失败。

## 目标

- 设置窗口的快捷键字段不能接受普通文本输入、粘贴、删除等任意编辑。
- 用户聚焦快捷键字段后，通过实际按键组合捕获快捷键并自动填入规范文本。
- 支持的主键为字母、数字和 `F1` 到 `F24`。
- 支持的修饰键为 `Ctrl`、`Alt`、`Shift`、`Win`。
- 单独按修饰键不更新快捷键。
- 快捷键必须包含至少一个修饰键，避免单个字母或数字变成全局热键。
- 保存设置时校验快捷键并保存规范化文本，例如 `Ctrl+Alt+T`。
- 保存后新快捷键立即生效，不需要重启应用。
- 保存失败或快捷键注册失败时给出用户可理解的错误提示，并保持设置窗口打开。

## 非目标

- 不实现 `Ctrl+C+C`、双击 `Ctrl`、连续按键序列或低级键盘 hook。
- 不支持鼠标按键作为快捷键。
- 不支持任意标点符号或媒体键作为主键。
- 不重做设置窗口整体布局。
- 不改变快捷键触发后的取词和翻译流程。

## 交互设计

快捷键控件仍放在设置窗口顶部的全局设置区，但行为从普通文本框改为只读捕获框。

用户点击或 Tab 聚焦该控件后：

- 按 `Ctrl`、`Alt`、`Shift`、`Win` 中的任意组合时，不改变文本。
- 在修饰键组合下按字母、数字或功能键时，控件文本更新为规范快捷键。
- 不带修饰键按字母、数字或功能键时，不更新文本。
- 按 `Esc` 不写入快捷键，保持设置窗口现有取消或关闭语义。
- 普通字符输入、粘贴、Backspace、Delete 等不会直接修改控件文本。

控件显示始终使用 `Hotkey::to_string()` 的格式：修饰键顺序为 `Ctrl`、`Alt`、`Shift`、`Win`，最后是主键。

## 数据和校验设计

现有 `Hotkey` 继续作为快捷键解析和展示的核心类型。需要扩展它以覆盖本次捕获范围：

- `KeyCode::Char` 继续支持字母。
- 新增或扩展数字主键支持，显示为 `0` 到 `9`。
- `KeyCode::Function` 继续支持 `F1` 到 `F24`。

保存设置时，`save_settings_from_window` 读取快捷键控件文本后先解析为 `Hotkey`，再把 `to_string()` 写入 `SettingsProfileDetailUpdate.hotkey`。这样配置文件不会保存未规范化或非法文本。

如果解析失败，保存失败并显示错误提示。设置文件不写入，设置窗口保持打开。

如果解析成功但没有任何修饰键，保存同样失败并显示错误提示。该校验可以放在 `Hotkey` 本身，也可以放在设置保存路径；实现时应优先选择能让测试覆盖最清晰的位置。

## 运行期生效设计

主消息循环把全局快捷键注册状态从启动时的一次性 `_registered` 改成可替换状态。

收到 `WM_SETTINGS_SAVED` 后：

1. 从配置文件重新加载设置。
2. 比较旧快捷键和新快捷键。
3. 如果未变化，只更新运行时设置和翻译窗口配置列表。
4. 如果变化，先解析新快捷键并尝试注册。
5. 注册成功后替换旧注册对象，旧快捷键被注销，新快捷键立即生效。
6. 注册失败时保留旧注册对象，继续使用旧快捷键，并把配置中的快捷键回滚为旧值。

注册失败时必须给用户明确提示，例如“快捷键注册失败，请换一个组合键；当前仍使用原来的快捷键”。提示内容应包含底层错误信息或冲突原因的摘要，便于用户知道是组合键不可用而不是保存按钮失效。

## 错误处理

设置窗口保存阶段：

- 非法快捷键：显示保存失败提示，说明快捷键格式不支持或必须包含普通按键。
- 空快捷键：显示保存失败提示，不写配置。
- 没有修饰键的快捷键：显示保存失败提示，说明快捷键必须包含 `Ctrl`、`Alt`、`Shift` 或 `Win`。

主程序重注册阶段：

- 新快捷键注册成功：无需额外提示。
- 新快捷键注册失败：显示错误提示，说明新快捷键没有生效，应用仍保留旧快捷键，并回滚配置文件里的快捷键字段。
- 配置重载失败：记录日志，保持现有运行状态。

错误提示需要面向普通用户，不暴露实现术语作为唯一说明。底层错误可以作为补充信息放在同一提示里。

## 测试设计

新增或更新单元测试覆盖：

- `Hotkey` 能解析和规范化数字主键。
- 没有修饰键的快捷键会被拒绝。
- 捕获逻辑能把修饰键状态和字母、数字、功能键转换成快捷键文本。
- 只按修饰键不生成快捷键。
- 不带修饰键按主键不生成快捷键。
- 不支持的按键不生成快捷键。
- 保存设置会把快捷键规范化后写入 `settings.hotkey`。
- 保存设置遇到非法快捷键时返回错误。
- 运行期快捷键切换逻辑在注册新快捷键成功时采用新值。
- 运行期快捷键切换逻辑在注册失败时保留旧值并返回可提示给用户的错误。

Windows UI 层仍需要人工验证：

- 点击快捷键框后按 `Ctrl+Alt+T`，框内显示 `Ctrl+Alt+T`。
- 尝试粘贴或输入普通文本不会污染快捷键。
- 保存后无需重启即可用新快捷键触发翻译。
- 输入冲突快捷键并保存时出现清楚的错误提示。
- 注册冲突后配置仍保留旧快捷键。

## 验收

- 设置窗口不能手动输入任意快捷键文本。
- 用户通过实际按键组合修改快捷键。
- 非法或冲突快捷键有明确错误提醒。
- 保存成功后新快捷键立即生效。
- 新快捷键注册失败时旧快捷键仍可用。
- `cargo test` 通过。
