# v0.1.2 开启自启和版本显示设计

## 背景

ait 是一个常驻系统托盘的 Windows 划词翻译工具。用户安装后，如果每次都需要手动启动，容易忘记工具存在，也会降低日常使用频率。v0.1.2 的目标是补齐最基础的常驻体验：用户可以在设置里开启自启，并且能在应用内看到当前版本号，方便反馈问题。

## 目标

- 在设置窗口增加一个复选框，文案为 `开启自启`。
- 自启开关默认关闭，只有用户主动勾选并保存后才启用。
- 点击设置窗口的 `保存` 后，自启状态才写入系统；点击 `取消` 不改变系统自启状态。
- 在设置窗口显示当前版本号，例如 `ait v0.1.2`。
- 不把自启状态写入 `settings.json`，设置窗口打开时读取 Windows 当前真实自启状态。

## 非目标

- 不做安装器层面的开机启动选项。
- 不做自动更新。
- 不做代码签名。
- 不做管理员级别的全用户自启。

## 自启实现

使用当前用户的 Windows Run 注册表项：

`HKCU\Software\Microsoft\Windows\CurrentVersion\Run`

条目名称使用：

`ait`

条目值使用当前运行的 `ait.exe` 路径。保存时如果用户勾选 `开启自启`，写入该值；如果用户取消勾选，删除该值。

该方案不需要管理员权限，符合个人桌面工具的使用场景。因为状态来自系统注册表本身，即使用户通过任务管理器或注册表手动关闭自启，设置窗口下次打开时也能显示真实状态。

## 设置窗口行为

设置窗口打开时：

- 读取现有 `AppSettings`，用于热键、翻译配置等已有设置。
- 读取 Run 注册表项，生成 `auto_start_enabled` 视图状态。
- 使用 `CARGO_PKG_VERSION` 或等价编译期版本常量生成版本显示文本。

设置窗口保存时：

- 按现有逻辑保存热键和翻译配置。
- 按复选框状态同步 Run 注册表项。
- 如果注册表操作失败，显示错误消息，并保持设置窗口打开。

设置窗口取消时：

- 关闭窗口。
- 不保存 `settings.json`。
- 不修改 Run 注册表项。

## UI 设计

设置窗口顶部保留现有快捷键设置。

在全局设置区域增加：

- 复选框：`开启自启`

版本号显示放在窗口底部或保存按钮附近，文本格式：

`ait v0.1.2`

版本号只是静态信息，不需要可编辑，也不需要复制按钮。

## 模块边界

新增一个专门处理自启的模块，例如 `src/startup.rs`：

- `is_auto_start_enabled() -> Result<bool>`
- `set_auto_start_enabled(enabled: bool) -> Result<()>`
- `auto_start_entry_name() -> &'static str`

Windows 实现读写 HKCU Run 注册表项。非 Windows 构建可以返回 `Ok(false)` 或空实现，保持测试和跨平台编译友好。

设置窗口只依赖这些函数，不直接拼注册表路径或调用 Windows API。

## 配置兼容性

不修改 `AppSettings` 结构，不新增 `settings.json` 字段，因此不需要配置迁移。

已有用户升级到 v0.1.2 后：

- 配置文件继续按旧格式读取。
- 自启默认按系统注册表真实状态显示。
- 如果用户从未设置过自启，默认显示未勾选。

## 测试策略

优先测试可纯函数验证的行为：

- 设置视图模型包含 `auto_start_enabled`。
- 设置视图模型包含版本显示文本，格式为 `ait vX.Y.Z`。
- 保存更新结构包含自启状态。
- `settings.json` 序列化不新增自启字段。

注册表读写逻辑应通过小边界封装测试：

- Run 项名称固定为 `ait`。
- 生成的启动命令指向当前 exe 路径。
- 开启时写入条目，关闭时删除条目。

如果直接读写真实 Run 注册表不适合单元测试，应把注册表访问包装成可替换边界，在单元测试里用内存实现验证逻辑。

## 发布说明

v0.1.2 发布说明应包含：

- 新增设置项：`开启自启`。
- 设置窗口显示当前版本号。
