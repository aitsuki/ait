# Win32 自绘按钮设计

## 背景

当前翻译窗口和设置窗口使用原生 Win32 `BUTTON` 控件。虽然已经统一了 UI 字体，但按钮仍保留系统默认边框、背景和按压效果，整体观感偏旧。为了逐步现代化界面，本阶段只处理按钮，不同时改动输入框、列表框、下拉框或窗口背景。

## 目标

- 使用 Win32 owner-draw 方式自绘应用内按钮。
- 保留现有按钮的窗口句柄、点击命令、Tab 导航和基本键盘行为。
- 区分主按钮和次按钮，让主要操作更清晰。
- 覆盖翻译窗口和设置窗口内现有按钮。
- 为后续输入框、列表框等控件自绘建立可复用的样式和绘制入口。

## 非目标

- 不引入第三方 UI 框架。
- 不重写窗口布局。
- 不实现自定义按钮窗口类。
- 不改动输入框、列表框、下拉框、复选框或窗口背景。
- 不新增图标按钮或动画。

## 按钮角色

主按钮用于页面或窗口中的主要确认动作：

- `翻译`
- `保存`

次按钮用于普通操作、辅助操作或取消动作：

- `有新版本`
- `新增`
- `删除`
- `设为默认`
- `显示`
- `隐藏`
- `取消`

按钮角色由创建按钮时显式传入，不依赖按钮文本判断。这样后续文案调整、翻译或复用按钮时不会改变样式语义。

## 视觉风格

按钮采用“安静蓝色”方向，整体保持工具型应用的克制感。

主按钮：

- 正常态：蓝色背景、白色文字、无明显外边框。
- 悬停态：背景略加深。
- 按下态：背景继续加深，并保持文字清晰。
- 禁用态：浅灰背景、灰色文字，不显示蓝色强调。

次按钮：

- 正常态：白色背景、浅灰边框、深色文字。
- 悬停态：极浅蓝灰背景、边框略加深。
- 按下态：浅蓝灰背景。
- 禁用态：浅灰背景、浅灰边框、灰色文字。

所有按钮：

- 使用 6px 左右圆角。
- 文字居中。
- 继续使用现有 `Microsoft YaHei UI` 统一字体。
- 焦点态使用细描边或虚线焦点框，保证键盘导航可见。

## 技术方案

继续创建 Win32 `BUTTON` 控件，但创建时增加 `BS_OWNERDRAW` 样式。父窗口在 `WM_DRAWITEM` 中接收绘制请求，根据控件 id 找到按钮角色并绘制。

新增一个 Windows-only 的按钮样式模块，例如 `src/ui/button.rs`：

- 定义 `ButtonRole`，包含 `Primary` 和 `Secondary`。
- 提供按钮绘制入口，接收 `DRAWITEMSTRUCT` 和角色。
- 根据 `itemState` 判断 normal、pressed、disabled、focus 等状态。
- 使用 GDI 绘制圆角背景、边框和文字。
- 暴露纯逻辑函数描述不同角色和状态对应的颜色，便于测试。

翻译窗口和设置窗口各自保留现有 `create_button` 辅助函数，但增加角色参数或提供 `create_primary_button` / `create_secondary_button` 小包装。窗口过程新增 `WM_DRAWITEM` 分支，把按钮绘制转发给统一模块。

## 交互和状态

点击行为继续走现有 `WM_COMMAND` 流程，不改变业务逻辑。

按钮需要覆盖以下状态：

- 正常
- 鼠标悬停
- 鼠标按下
- 禁用
- 键盘焦点

Win32 owner-draw 默认不会自动提供完整 hover 重绘。实现时需要在按钮子类或父窗口鼠标跟踪中记录 hover 状态，并在进入/离开时触发按钮重绘。第一版只为 owner-draw 按钮安装轻量子类，用于处理 `WM_MOUSEMOVE`、`WM_MOUSELEAVE`、`WM_NCDESTROY`，不改变点击消息。

## 错误处理

按钮创建失败沿用现有 `AppError::Windows`。

绘制阶段不返回业务错误。绘制函数应尽量容错：如果 GDI 对象创建失败，退回到简单矩形背景和文本绘制，避免窗口无法交互。

## 测试

自动化测试覆盖纯逻辑部分：

- 主按钮和次按钮角色映射正确。
- 各状态对应的颜色选择稳定。
- 已知按钮 id 能映射到预期角色。
- 未知按钮 id 不触发自绘成功路径。

现有窗口布局测试保持不变，因为本阶段不改变按钮位置和尺寸。

手动验证：

- 打开翻译窗口，确认 `翻译` 为主按钮，`有新版本` 为次按钮。
- 打开设置窗口，确认 `保存` 为主按钮，其它按钮为次按钮。
- 检查 hover、pressed、disabled、focus 状态。
- 检查中文文字在按钮内居中且不截断。

## 风险

- owner-draw 按钮需要手动处理 hover 重绘，否则视觉反馈会不完整。
- GDI 圆角和文字绘制在高 DPI 下需要使用当前控件尺寸计算，不能写死像素。
- 如果遗漏 `WM_DRAWITEM` 的默认处理，可能影响非按钮 owner-draw 控件。第一版只处理已知按钮 id，其它情况交回默认窗口过程。
