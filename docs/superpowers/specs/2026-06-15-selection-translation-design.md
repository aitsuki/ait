# 划词翻译软件首版设计规格

日期：2026-06-15

## 目标

首版目标是一个 Windows 10+ 的轻量划词翻译工具，形态为托盘常驻、无主界面。用户在任意应用中选中文本后，按固定全局快捷键，例如 `Ctrl+Alt+E`，软件通过剪贴板复制策略取到选中文本，弹出翻译窗口，默认调用内置 Google 非官方免 Key 翻译，完整返回后一次性显示结果。

本项目强调轻量和 Windows 原生能力，主实现采用 Rust + Win32。快捷键序列和取词能力必须优先参考成熟开源项目的行为与边界，详见 `docs/research/open-source-shortcuts-and-text-capture.md`。

## 首版范围

### 包含

- 托盘常驻和托盘菜单：翻译、设置、退出。
- 固定全局快捷键，默认 `Ctrl+Alt+E`，支持在设置里修改。
- 剪贴板优先取词：发送复制命令读取文本，并尽力恢复原文本剪贴板。
- 翻译窗口：可移动、可调整大小、可关闭、默认保持显示。
- 内置 Google 非官方免 Key 翻译，作为开箱默认翻译提供方。
- OpenAI 兼容接口配置，作为可选翻译提供方。
- 最小设置窗口：翻译提供方、OpenAI 兼容接口配置、快捷键、剪贴板取词开关。
- 配置文件：保存普通设置；API Key 用 DPAPI 保护。
- 翻译结果首版纯文本显示；预留 Markdown 渲染开关和内部接口，但不实现渲染。
- 日志：记录快捷键、取词策略、接口错误和剪贴板恢复失败。

### 不包含

- UI Automation 取词。
- OCR。
- 翻译历史和收藏。
- 流式输出。
- Anthropic、官方 Google Cloud Translation、Gemini 或任意 HTTP 模板。
- 失焦自动隐藏。
- 完整恢复图片、文件列表、富文本等复杂剪贴板格式。

## 技术方案

采用 Rust + Win32 原生轻量应用：

- 主程序使用 Rust。
- Win32 能力通过 `windows-rs` 调用。
- 不使用 Electron、WebView2、Flutter、Qt 等高运行时成本方案作为主 UI。
- 托盘、窗口、快捷键、剪贴板、DPAPI、DPI 和多显示器行为贴近 Win32 实现。

不采用 Rust 核心 + WebView 设置页，因为不符合极致轻量目标。不采用 AutoHotkey 或脚本层作为产品架构；它们只作为快捷键序列行为参考。

## 模块边界

### `app`

负责程序生命周期、单实例、消息循环、托盘初始化、模块装配。这里不放业务逻辑，只协调启动、退出和全局事件分发。

### `hotkey`

负责全局快捷键注册和解析。首版只必须支持 `RegisterHotKey` 这类固定组合键；为未来 `Ctrl+C+C`、双击 `Ctrl` 预留事件状态机接口，但首版不默认启用 low-level keyboard hook。实现时参考 `global-hotkey`、AutoHotkey、GoldenDict 的行为和边界。

### `capture`

负责取选中文本。首版只有剪贴板复制策略：保存当前文本剪贴板，发送 `Ctrl+C`，等待剪贴板变化，读取文本，恢复原文本剪贴板。它对外只返回 `CapturedText` 或明确错误，不直接弹 UI、不调用翻译接口。

### `translator`

负责翻译提供方调用。输入为原文、目标语言、提供方配置；输出为完整翻译文本或结构化错误。首版不流式。

首版提供两个适配器：

- `GoogleFreeTranslator`：内置默认提供方，参考开源项目对 Google Translate 非官方网页端点的使用方式，不需要 API Key。
- `OpenAiCompatibleTranslator`：用户配置 Base URL、API Key 和 Model 后使用，兼容 OpenAI Chat Completions 风格接口。

`GoogleFreeTranslator` 不是 Google Cloud Translation 官方 API。它可能受到限流、403/429、网页端点格式变化或服务策略变化影响。失败时应给出可理解错误，并提示用户切换到 OpenAI 兼容接口。

### `ui`

负责托盘、翻译窗口、设置窗口。翻译窗口显示原文和译文，支持复制、关闭、调整大小。设置窗口写入配置，不直接持有 API Key 明文。

### `config`

负责加载和保存普通配置，例如快捷键、Base URL、Model、窗口尺寸、剪贴板取词开关。API Key 字段只存 DPAPI 保护后的密文。

### `secret`

封装 DPAPI 加密/解密。其他模块不直接碰 Win32 DPAPI 调用。

### `log`

负责轻量本地日志。至少记录取词失败、快捷键注册失败、接口错误、剪贴板恢复失败。

核心原则：快捷键触发只发出“请求翻译选区”的命令；取词模块只负责拿文本；翻译模块只负责请求模型；UI 只负责展示和设置。

## 主流程

1. 程序启动后进入单实例检查，初始化配置、托盘、快捷键和日志。
2. 用户在任意应用中选中文本，按默认快捷键 `Ctrl+Alt+E`。
3. `hotkey` 发出 `TranslateSelection` 命令。
4. `capture` 执行剪贴板取词：
   - 读取并保存当前剪贴板文本内容。
   - 发送复制命令。
   - 等待短时间窗口内出现新的文本剪贴板内容。
   - 如果拿到非空文本，就恢复之前保存的文本剪贴板。
   - 如果失败，也尽力恢复之前保存的文本剪贴板。
5. 取词成功后打开或复用翻译窗口，先显示原文和加载状态。
6. `translator` 调当前翻译提供方，默认使用内置 Google 非官方免 Key 翻译；如果用户切换为 OpenAI 兼容接口，则调用已配置接口。
7. 翻译完整返回后一次性更新译文。
8. 用户可以复制译文、关闭窗口，或打开设置调整接口和快捷键。

## 错误处理

- 快捷键注册失败：托盘提示并在设置中标出冲突，用户可改快捷键。
- 没取到文本：打开翻译窗口的手动输入状态，让用户粘贴或输入文本。
- 剪贴板恢复失败：不弹打扰式错误，但记录日志；必要时托盘提示。
- API Key 缺失：直接打开设置窗口并定位到接口配置。
- 内置 Google 非官方免 Key 翻译失败：翻译窗口显示错误摘要，提示可重试或切换到 OpenAI 兼容接口。
- 接口请求失败：翻译窗口显示错误摘要和“重试”按钮。
- 接口超时：取消本次请求，显示超时错误，可重试。
- 配置损坏：备份损坏文件，恢复默认配置，并提示用户重新配置接口。

关键约束：

- 首版只承诺恢复文本剪贴板，不承诺恢复图片、文件列表、富文本等格式。
- 发送复制命令后要有短暂等待和防抖，避免读到旧剪贴板。
- 如果用户关闭翻译窗口，正在进行的请求应取消或忽略结果，不能把结果写回已关闭窗口。
- 日志不能记录 API Key；默认也不记录完整待翻译文本，只记录 provider、状态码、文本长度和错误类型。

## UI 行为

### 翻译窗口

- 默认在当前屏幕中央弹出；后续可优化到鼠标或选区附近。
- 窗口默认保持显示，不因失焦自动隐藏。
- 支持拖动、调整大小、关闭，并记住上次尺寸。
- 内容分为原文区和译文区。
- 原文区显示本次捕获的文本，可折叠或限制高度，避免长文本挤掉译文。
- 译文区首版使用纯文本显示，支持选择和复制。
- 按钮包括：复制译文、重试、打开设置、关闭。
- Markdown 开关可以先显示为禁用或隐藏；内部保留配置字段，但首版不渲染。

### 托盘

- 托盘图标常驻。
- 菜单包含：翻译剪贴板/选区、设置、查看日志、退出。
- 快捷键注册失败或接口未配置时，可以通过托盘提示告知用户。

### 设置窗口

接口配置：

- 默认翻译提供方：内置 Google 非官方免 Key 翻译 或 OpenAI 兼容接口。
- 名称。
- Base URL。
- API Key。
- Model。
- 请求超时。
- 测试连接按钮。
- 内置 Google 非官方免 Key 翻译不显示 API Key 输入框，但设置界面要提示它不是官方 Google Cloud Translation API，稳定性不保证。

快捷键：

- 默认 `Ctrl+Alt+E`。
- 支持录制新的固定组合键。
- 首版不提供 `Ctrl+C+C` 和双击 `Ctrl` 的 UI 开关，只在架构中预留。

取词：

- 启用剪贴板取词。
- 取词失败时打开手动输入窗口。
- 提示“首版只恢复文本剪贴板，不保证恢复图片/文件/富文本剪贴板”。

常规：

- 开机启动首版先不做。
- 日志位置和打开日志按钮可以做。

## 配置数据

`settings.json` 保存普通配置：

- 快捷键。
- 默认翻译提供方。
- OpenAI 兼容接口的 Base URL、Model、超时。
- DPAPI 加密后的 API Key 字段。
- 窗口尺寸。
- 是否启用剪贴板取词。
- 取词失败是否打开手动输入。
- 预留 Markdown 渲染开关，首版默认关闭且不生效。

API Key 不以明文落盘。配置损坏时备份旧文件并生成默认配置。

## 测试重点

### `hotkey`

- 默认快捷键解析。
- 快捷键冲突时返回明确错误。
- 为未来序列快捷键保留状态机测试入口。

### `capture`

- 有选中文本时能取到文本。
- 没有选中文本时返回失败。
- 原文本剪贴板能恢复。
- 空文本、超时、剪贴板被占用时有明确错误。

### `translator`

- 内置 Google 非官方免 Key 翻译请求和响应解析正确。
- Google 非官方端点返回 403、429、格式变化或网络失败时有结构化错误。
- OpenAI 兼容请求体正确。
- API Key 缺失、401、超时、网络失败有结构化错误。
- 不记录 API Key。

### `config` / `secret`

- 配置读写。
- DPAPI 加密解密。
- 损坏配置恢复。

### 手工验证

- Notepad、浏览器、PDF 阅读器、聊天软件输入框里选中文本后触发。
- 不同 DPI、多显示器、窗口关闭时请求返回。
- 剪贴板中已有文本时是否恢复。

## 原型顺序

1. `hotkey` + 托盘最小程序：启动后注册 `Ctrl+Alt+E`，触发时写日志。
2. `capture` 剪贴板取词原型：触发后取选中文本并恢复原文本剪贴板。
3. 翻译窗口原型：显示原文和手动填充的译文，支持复制和关闭。
4. 内置 Google 非官方免 Key 翻译调用：无 API Key 完成真实翻译，失败时显示可理解错误。
5. OpenAI 兼容接口调用：设置接口后完成真实翻译。
6. 最小设置窗口：写入配置、DPAPI 保存 API Key、测试连接、切换默认翻译提供方。
7. 整合错误处理和日志。
8. 对照开源项目复核行为：GoldenDict、Crow Translate、Pot Desktop、Capture2Text、Text Grab、AutoHotkey、translate-shell、py-googletrans、deep-translator，特别确认快捷键、取词和 Google 非官方免 Key 翻译行为没有偏离既定约束。
