# 翻译窗口操作行为调整设计

日期：2026-06-16

## 背景

当前翻译窗口底部包含“复制译文”“重试”“设置”三个按钮。其中“重试”会发送托盘的“翻译剪贴板/选区”命令，导致重新执行取词到翻译的完整流程。用户期望窗口内操作只围绕当前窗口内容，设置入口由系统托盘右键菜单承担，译文复制由文本选区完成。

托盘右键菜单当前包含“翻译剪贴板/选区”。用户期望该入口改为“显示翻译窗口”，仅显示窗口，不触发取词和翻译动作。

## 目标

- 翻译窗口不再显示“设置”按钮。
- 翻译窗口不再显示“复制译文”按钮。
- 翻译窗口将“重试”改为“翻译”。
- “翻译”按钮只翻译原文区当前文本，不重新取词。
- 托盘菜单将“翻译剪贴板/选区”改为“显示翻译窗口”，并且只显示窗口，不触发取词或翻译。
- 原文区和译文区支持常见文本选区交互：双击选中单词、三连击选中段落、`Ctrl+A` 全选。
- 翻译窗口在前台或获得焦点时，按 `Esc` 快速关闭窗口。

## 非目标

- 不新增 OCR、截图识别或区域识别能力。
- 不改变全局热键的取词翻译行为。
- 不改变系统托盘右键菜单中的设置入口。
- 不新增独立复制按钮或复制快捷键。

## 交互设计

翻译窗口底部只保留一个“翻译”按钮。用户可以编辑原文区文本，点击“翻译”后应用直接翻译原文区内容，并更新译文区。该动作不会调用取词服务，不会读取当前选区，也不会发送复制快捷键。

如果原文区为空或只有空白字符，点击“翻译”不发起翻译请求，窗口显示错误状态，提示原文为空。

译文区仍为只读文本区，但必须允许用户选择文本并复制。原文区和译文区都应响应双击、三连击和 `Ctrl+A`，使用户可以直接使用系统文本选择和复制能力。

托盘右键菜单中的入口改为“显示翻译窗口”。点击该入口时，显示翻译窗口并保留窗口已有内容，不进入“正在取词”状态，不清空原文或译文，不发起翻译请求。如果窗口已经显示，则将窗口带到最顶部并获取焦点。

当翻译窗口本身或其子控件处于焦点链路内时，按 `Esc` 隐藏翻译窗口。该动作不退出应用，不清空窗口状态。

## 架构设计

`TranslationWorkflow` 增加按文本翻译的可测试路径，例如 `translate_text_with_observer(source_text, target_lang, observer)`。现有 `translate_selection_with_observer` 在捕获文本后复用该路径。这样“选区翻译”和“原文区翻译”共享翻译逻辑，但取词职责只存在于选区翻译路径。

Windows 消息循环新增一个窗口命令用于“翻译当前原文”。翻译窗口过程读取原文编辑框内容后发送该命令，应用层收到后调用按文本翻译路径。该设计避免在窗口过程里直接构造翻译器或读取配置。

托盘菜单命令从“翻译选区”语义调整为“显示翻译窗口”语义。应用层收到该命令后只调用翻译窗口的显示方法，不调用 `perform_translation`。

编辑框选择行为由翻译窗口模块统一处理。优先使用标准 Windows 编辑控件已有能力；若默认控件不能满足 `Ctrl+A` 或三连击段落选择，则通过子类过程补齐。补齐逻辑仅作用于翻译窗口的原文区和译文区。

## 数据流

全局热键：

1. 用户按全局热键。
2. 应用执行现有取词流程。
3. 取词成功后显示原文并翻译。
4. 翻译成功后更新译文。

窗口“翻译”按钮：

1. 用户编辑或保留原文区内容。
2. 点击“翻译”。
3. 翻译窗口读取原文区文本。
4. 应用直接翻译该文本。
5. 译文区更新。

托盘“显示翻译窗口”：

1. 用户点击托盘菜单。
2. 应用显示翻译窗口；如果窗口已经显示，则将窗口带到最顶部并获取焦点。
3. 窗口内容保持不变。
4. 不取词，不翻译。

## 错误处理

窗口“翻译”按钮遇到空原文时，显示本地错误状态，不调用翻译提供方。

翻译提供方失败时，复用现有错误展示逻辑，在状态栏显示错误信息，保留原文区内容，译文区保持可选择。

显示翻译窗口失败时记录日志，不触发取词或翻译兜底。

## 测试设计

新增或调整工作流测试，覆盖：

- 按文本翻译不会调用 capture。
- 选区翻译仍按 started、capture、source、translate、result 顺序通知。
- 空原文不调用 translator，并返回明确错误。

新增或调整窗口行为测试，覆盖：

- 托盘“显示翻译窗口”对应的应用动作不是翻译选区。
- `Esc` 在翻译窗口焦点内对应隐藏窗口动作。
- `Ctrl+A` 在编辑框焦点内对应全选动作。

Windows 控件的双击和三连击行为需要至少做手工验证，因为它依赖实际系统编辑控件消息行为。手工验证项写入现有 Windows MVP 检查清单。
