# 设置与翻译窗口修复设计

## 背景

多接口配置改造后，设置窗口和翻译窗口暴露出一组交互问题：设置保存会关闭窗口，全局设置混在接口详情里，内置配置标记过多，Google 配置仍显示不适用的接口表单，翻译窗口配置下拉无法正常展开，翻译期间窗口会卡死。

这些问题都集中在设置窗口布局、配置详情视图模型、Win32 下拉框尺寸和主线程同步翻译路径上。目标是在保持现有配置数据结构和多接口能力的前提下，修正用户可见行为。

## 目标

- 设置窗口点击“保存”后保存配置并保持窗口打开。
- 设置窗口分为上下两块：上方为全局设置，下方为接口配置，中间用分割线隔开。
- 全局设置包含快捷键和复制等待毫秒。
- 接口配置不再显示快捷键、复制等待毫秒和供应商选择。
- 配置列表不显示“内置”备注，只保留名称和“默认”标记。
- 选中内置配置时删除按钮不可点击。
- Google 配置不可编辑，不显示 Base URL、模型、API Key、超时等网络表单。
- Google 免 Key 声明只在选中 Google 时显示。
- 翻译窗口配置下拉框可以正常显示配置列表。
- 翻译请求不阻塞 UI 线程，正在翻译时窗口仍可响应移动、关闭、重绘等消息。

## 非目标

- 不重做配置文件格式。
- 不引入供应商原生协议适配。
- 不实现远程模型列表拉取。
- 不改变已有内置非 Google 配置的可编辑能力。
- 不改变翻译结果展示样式。

## 设置窗口设计

设置窗口使用固定尺寸 Win32 窗口，布局调整为两段式。

上方全局设置区：

- 快捷键编辑框。
- 复制等待毫秒编辑框。

中间绘制水平分割线，分隔全局设置和接口配置。

下方接口配置区：

- 左侧配置列表。
- 左侧下方保留“新增”“删除”“设为默认”按钮。
- 右侧显示当前配置详情。

配置列表标签规则：

- 默认配置显示为 `名称（默认）`。
- 非默认配置显示为 `名称`。
- 不显示“内置”。

删除按钮规则：

- 选中内置配置时禁用。
- 选中自定义配置时启用。
- 底层 `delete_profile` 仍保留内置配置不可删除的校验。

保存按钮规则：

- 保存成功后不销毁设置窗口。
- 保存成功后刷新列表、当前详情、删除按钮和 Google 声明显示状态。
- 保存成功后继续发送 `WM_SETTINGS_SAVED` 给主窗口，让运行时重新加载设置并刷新翻译窗口下拉框。
- 保存失败时显示错误，并保持窗口打开。

取消按钮规则：

- 仍关闭设置窗口，不保存未保存修改。

## 接口详情设计

接口详情不显示供应商选择。profile 的 provider 由配置本身决定，用户不能在窗口里把一个配置切换成另一个供应商类型。

Google 配置：

- 视为只读配置。
- 不显示 Base URL、模型、API Key 和超时秒数表单。
- 不显示供应商选择。
- 显示 Google 免 Key 声明。
- 名称不允许编辑。
- 保存时不会写入 Google 网络字段。

非 Google 配置：

- 显示名称、Base URL、模型、API Key、超时秒数。
- 不显示 Google 免 Key 声明。
- 内置非 Google 配置仍允许编辑这些字段，用于填写 API Key 或覆盖默认模型、Base URL。
- 自定义配置同样允许编辑。

保存数据时：

- 全局设置从上方控件读取。
- 当前 profile 从接口详情控件读取。
- Google profile 保持 provider 为 Google，并清空网络字段。
- 非 Google profile 保持原 provider，不从 UI 读取 provider。

## 翻译窗口下拉框设计

翻译窗口配置下拉框当前创建后被布局函数移动成 26 像素高，导致 Win32 combobox 的下拉区域高度也被压缩。

修复方式：

- 布局模型区分控件显示高度和 combobox 总高度。
- 视觉上仍显示 26 像素高。
- `CreateWindowExW` 和 `MoveWindow` 对 combobox 使用足够的总高度，例如 180 到 220 像素，让展开列表有可用空间。
- 其他控件继续使用现有布局规则。

## 异步翻译设计

主消息循环不再直接执行阻塞翻译请求。翻译动作拆成“启动任务”和“处理结果”两个阶段。

启动任务：

- UI 线程读取当前 source text 或触发选区捕获前的必要状态。
- 显示“正在取词...”或“正在翻译...”状态。
- 创建当前 settings、active profile 和必要参数的快照。
- 在线程中执行捕获和翻译。

处理结果：

- 后台线程完成后向翻译窗口或主消息循环投递完成消息。
- UI 线程收到完成消息后更新译文或错误状态。
- 成功时显示译文和“翻译完成”。
- 失败时保留原文，显示错误。

并发规则：

- 第一版允许后发起的翻译覆盖先完成的翻译结果，但应避免崩溃和窗口卡死。
- 可以为任务分配递增 id；如果实现成本低，UI 只接受最新任务结果。

错误处理：

- API Key 缺失、网络错误和空原文仍通过现有错误文案显示。
- 异步任务中的错误不应 panic 到 UI 线程。
- 后台线程不能直接操作 Win32 控件，只能通过消息把结果交回 UI 线程。

## 测试

单元测试：

- 设置列表标签不包含“内置”。
- 默认配置标签只包含“默认”。
- Google detail 标记为只读、不可删除、不显示网络字段、显示免 Key 声明。
- 非 Google detail 可编辑，显示网络字段，不显示免 Key 声明。
- 保存 detail update 不允许从 UI 改变 provider。
- 翻译窗口布局给 combobox 保留足够下拉高度。

窗口逻辑或可分离逻辑测试：

- 保存成功动作不请求关闭窗口。
- 选中内置配置时删除按钮状态为禁用。
- 翻译请求启动后 UI 状态进入 loading，完成消息再更新结果。

人工验证：

- 打开设置窗口，保存后窗口仍打开。
- Google 配置只显示免 Key 信息和只读状态。
- OpenAI、Claude、Gemini、DeepSeek、自定义配置可编辑接口字段。
- 翻译窗口配置下拉能展开并显示所有配置。
- 使用网络较慢的翻译配置时，窗口仍可移动、关闭和重绘。

## 验收

- 用户列出的八项设置窗口问题全部修复。
- 翻译窗口下拉框可正常选择配置。
- 翻译期间窗口不再卡死。
- 现有配置文件可继续加载，旧测试继续通过。
- 新增测试覆盖主要视图模型、布局和保存行为。
